# CI Workflows

## Quality And Tests

Workflow: `.github/workflows/quality-and-test.yml`

Stages:

- formatting/lint checks (`ruff`, `black`, `isort`)
- static type checks (`mypy`)
- tests with coverage (`pytest --cov`)

## Publish

Workflow: `.github/workflows/publish.yml`

Trigger:

- tags matching `v*.*.*`

Release steps:

- build source + wheel
- publish to PyPI with `PYPI_TOKEN`

Keep release tags aligned with semantic versioning in your release process.
