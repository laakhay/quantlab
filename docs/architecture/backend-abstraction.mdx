# Backend Abstraction

## Protocol Contract

`Backend` defines a large surface area for:

- array construction and shape transforms
- arithmetic, reductions, and comparisons
- random generation and special functions
- linear algebra and device-aware operations

Implementations must match this contract to be swappable.

## Runtime Components

- `register(name, backend)`
- `set_default(name)`
- `get_backend(name=None)`
- `infer_backend(array)` / `infer_backend_from_arrays(*arrays)`
- `convert_array(array, to_backend, from_backend=None)`

## Wrapper Layer

`ArrayBackend` adds ergonomic and acceleration helpers:

- `grad`, `value_and_grad`
- `vmap`
- `jit`

The wrapper delegates numeric primitives to underlying concrete backend.
